text

    │
    ▼
OCR (processar_uploads.py)      ← push no uploads/
    │
    ├─ extrai apostas → commita apostas/*.json
    ├─ gera thumbnails → commita thumbnails/
    ├─ limpa imagens → commita remoção das imagens
    └─ verifica validações pendentes → envia push
    │
    ▼
4× Verificar * (um por jogo)    ← push nas apostas ou workflow_run
    │
    ├─ corre o verificador → commita resultados/*_recentes.json
    └─ e resultados/*_verificacoes.json
    │
    ▼
Detetar Novos Resultados        ← push nos recentes OU workflow_run dos verificadores
    │
    ├─ verifica validações pendentes
    ├─ gera notificações → commita notificacoes_ativas.json
    └─ envia push
    │
    ▼
Atualizar Estatísticas          ← workflow_run do Detetar
    │

Cada commit dispara um Pages build (porque tens o GitHub Pages ativo).

2. Problemas estruturais identificados
🔴 Problema 1: Concorrência de commits (race condition)
Onde dói:
Quatro verificadores correm em paralelo após um commit nas apostas. Cada um faz git stash, git pull, git stash pop, git add, git commit, git push. Se dois terminam ao mesmo tempo, um push é rejeitado (rejected, fetch first) e os resultados desse verificador perdem-se.

Evidência real (do teu log de ontem):

text
! [rejected] main -> main (fetch first)
Error: Process completed with exit code 1.
O OCR processou 4 imagens, conseguiu extrair 1 aposta, mas o push falhou. A aposta ficou "pendurada" e a imagem não foi limpa.

🔴 Problema 2: Imagens órfãs por push falhado
Quando o push do OCR falha:

A aposta extraída não chega ao repositório.

A imagem não é removida da pasta uploads/.

Na próxima execução, a imagem é processada novamente → gera duplicado.

🟡 Problema 3: Cadeia de disparos excessiva
Um único upload dispara:

OCR (1 workflow)

4 verificadores (4 workflows em paralelo)

Detetar Novos Resultados (1 workflow, mas disparado 4 vezes porque cada verificador aciona um workflow_run)

Atualizar Estatísticas (disparado múltiplas vezes)

Pages build (disparado a cada commit)

Resultado: dezenas de execuções para uma única foto, consumindo minutos de Actions e gerando conflitos.

🟡 Problema 4: Disparos múltiplos do Detetar Novos Resultados
O detectar_resultados.yml tem este gatilho:

yaml
workflow_run:
    workflows:
      - "Verificar Totoloto"
      - "Verificar M1lhão"
      - "Verificar Euromilhões"
      - "Verificar EuroDreams"
    types:
      - completed
Quando os 4 verificadores correm, o Detetar é disparado 4 vezes seguidas. Cada execução lê os mesmos ficheiros, gera as mesmas notificações e tenta fazer push. Além de desperdício, pode gerar conflitos de merge (como o que viste no notificacoes_ativas.json).

🟡 Problema 5: Estatísticas calculadas múltiplas vezes
O workflow de estatísticas é disparado por workflow_run do Detetar. Como o Detetar corre várias vezes, as estatísticas também correm várias vezes — e cada uma faz um commit se houver alterações.

🟢 Problema 6: Verificadores dependentes de workflow_run do scraper
Se o scraper falhar (site em baixo, timeout, etc.), os verificadores não correm automaticamente porque o workflow_run só dispara em caso de sucesso (conclusion == 'success'). Isto é bom para evitar verificações contra sorteios incompletos, mas significa que se o scraper falhar consistentemente, os resultados nunca são verificados até alguém executar manualmente.

3. Plano de reestruturação
🎯 Objetivos
Eliminar race conditions → um único commit por "ciclo de verificação".

Garantir entrega das apostas → OCR resiliente, sem perda de dados.

Reduzir execuções → de ~15-20 para 3-4 por upload.

Simplificar a manutenção → menos ficheiros YAML, fluxo linear.

📋 Passo 1: Unificar os 4 verificadores num só workflow
Ficheiro a criar: .github/workflows/verificar_tudo.yml

O que faz:

Uma única job que executa todos os scripts de verificação sequencialmente.

Faz um único commit com todos os resultados.

No final, chama os scripts de notificações e estatísticas (eliminando a necessidade do detectar_resultados.yml e atualizar_estatisticas.yml separados).

YAML proposto:

yaml
name: Verificar Tudo

on:
  push:
    paths:
      - 'dados/*.json'
      - 'apostas/*.json'
  workflow_dispatch:
  workflow_run:
    workflows:
      - "Atualizar Totoloto SC automaticamente"
      - "Atualizar EuroMilhões automaticamente"
      - "Atualizar EuroDreams automaticamente"
      - "Atualizar M1lhão automaticamente"
    types:
      - completed

concurrency:
  group: git-push
  cancel-in-progress: false

jobs:
  verificar:
    if: ${{ github.event_name != 'workflow_run' || github.event.workflow_run.conclusion == 'success' }}
    runs-on: ubuntu-latest
    timeout-minutes: 15
    permissions:
      contents: write

    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'

      - name: Instalar dependências
        run: pip install Pillow pywebpush

      # Executar todos os verificadores
      - name: Verificar Totoloto
        run: python scripts/verificar_totoloto.py

      - name: Verificar Euromilhões
        run: python scripts/verificar_euromilhoes.py

      - name: Verificar EuroDreams
        run: python scripts/verificar_eurodreams.py

      - name: Verificar M1lhão
        run: python scripts/verificar_milhao.py

      # Gerar notificações
      - name: Gerar notificações
        env:
          VAPID_PRIVATE_KEY: ${{ secrets.VAPID_PRIVATE }}
          VAPID_EMAIL: ${{ secrets.EMAIL_REMETENTE }}
        run: python scripts/gerar_notificacoes.py

      # Gerar estatísticas
      - name: Gerar estatísticas
        run: python scripts/gerar_estatisticas_completas.py

      # Commit único
      - name: Commit resultados
        run: |
          git config user.name "Bot Verificador"
          git config user.email "bot@github.com"
          
          git stash
          n=0
          until [ "$n" -ge 5 ]; do
             git pull origin main --rebase && break
             n=$((n+1))
             sleep 15
          done
          git stash pop || echo "Sem stash"
          
          git add resultados/
          if ! git diff --cached --quiet; then
            git commit -m "Verificação + notificações + estatísticas $(date +'%Y-%m-%d %H:%M')"
            n=0
            until [ "$n" -ge 5 ]; do
               git pull origin main --rebase && git push origin main && break
               n=$((n+1))
               sleep 15
            done
          else
            echo "Sem alterações"
          fi

📋 Passo 2: Tornar o push do OCR à prova de falhas
Alterar o ficheiro: .github/workflows/ocr.yml

O que muda:

Adicionar concurrency: git-push para não colidir com o Verificar Tudo.

Após processar cada imagem, movê-la para uma subpasta processadas/ dentro de uploads/, em vez de depender do push para a limpar.

Adicionar retry no push final.

Trecho a alterar (final do ficheiro):

yaml
concurrency:
  group: git-push
  cancel-in-progress: false

# ... (resto do workflow igual)

      # 🔄 NOVO: Mover imagens processadas em vez de apagar
      - name: Mover imagens processadas
        run: |
          mkdir -p uploads/processadas
          for img in uploads/*.png uploads/*.jpg uploads/*.jpeg; do
            [ -f "$img" ] && mv "$img" uploads/processadas/ || true
          done

      - name: Commit e push (com retry)
        run: |
          git config user.name "Bot OCR"
          git config user.email "bot@github.com"
          mkdir -p apostas thumbnails
          git add apostas/*.json thumbnails/ uploads/processadas/
          
          if ! git diff --cached --quiet; then
            git commit -m "OCR: dados, thumbnails e imagens processadas [skip ci]"
            n=0
            until [ "$n" -ge 5 ]; do
               git pull origin main --rebase && git push origin main && break
               n=$((n+1))
               sleep 20
            done
          fi
Benefício: Mesmo que o push falhe, as imagens já estão em processadas/ e não serão re-processadas.

📋 Passo 4: Ajustar o gatilho de "Verificar Tudo"
Atualmente, os verificadores disparam com push nos apostas/*.json. Isto significa que o próprio commit do OCR (que guarda as apostas) já dispara o Verificar Tudo. Mas o sorteio pode ainda não estar disponível. Isto é aceitável: o verificador simplesmente não encontrará o sorteio e não gerará resultados.

Após a unificação, o fluxo fica:

text

    │
    ▼
OCR (única job, commit único)
    │
    ▼
Verificar Tudo (única job, commit único)
    │
    └─ inclui notificações e estatísticas

🔧 Plano ajustado (com base no feedback)
As alterações são poucas mas importantes:

Usar concurrency apenas em workflows que fazem push, com o grupo repo-write (não git-push) e cancel-in-progress: false.

Não usar [skip ci] no OCR – assim o commit do OCR dispara o Verificar Tudo automaticamente.

Usar git stash -u para incluir ficheiros novos (untracked) e evitar conflitos silenciosos.

Adicionar paths-ignore no Verificar Tudo para que os commits de resultados não voltem a dispará‑lo.

Tornar os passos de verificação tolerantes a falhas (continue-on-error: true) para que todos corram mesmo se um jogo falhar.

Organizar a pasta uploads/processadas por data para evitar crescimento infinito.
